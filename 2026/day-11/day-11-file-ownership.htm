## Day 11 – File Ownership Challenge (chown & chgrp)

 

## Objective

The goal of Day 11 is to understand and practice **Linux file and directory ownership**, including:

- User ownership

- Group ownership

- Changing ownership using `chown` and `chgrp`

- Applying ownership changes recursively

- Validating ownership using `ls -l`

 

---

 

## Task 1: Understanding File Ownership

 

### Command Used

ls –l

### Ownership Breakdown

 

·        Owner → The user who owns the file (E.g. root)

·        Group → The group associated with the file (E.g. root)

 

### Difference Between Owner and Group

 

·        The owner has primary control over the file (read/write/execute permissions).

·        The group allows shared access to users belonging to that group.

·        This model is essential for team-based access control in Linux systems.

 

## Task 2: Basic chown Operations

 

### File Created

touch devops-file.txt

 

### Check Current Ownership

ls -l devops-file.txt

 

### Create Users (if not present)

sudo useradd tokyo

sudo useradd berlin

 

### Change File Owner

sudo chown tokyo devops-file.txt

sudo chown berlin devops-file.txt

 

### Verification

ls -l devops-file.txt

 

     

 

## Task 3: Basic chgrp Operations

### File Created

touch team-notes.txt

 

### Check Current Group

ls -l team-notes.txt

 

### Create Group

sudo groupadd heist-team

 

### Change File Group

sudo chgrp heist-team team-notes.txt

 

### Verification

ls -l team-notes.txt

 

    

 

 

## Task 4: Change Owner and Group Together

### File Created

touch project-config.yaml

 

### Change Owner and Group (Single Command)

sudo chown professor:heist-team project-config.yaml

 

### Directory Created

mkdir app-logs

 

### Change Directory Ownership

sudo chown berlin:heist-team app-logs

 

### Verification

ls -ld project-config.yaml app-logs

          

    

 

 

## Task 5: Recursive Ownership Changes

 

### Directory Structure Created

mkdir -p heist-project/vault

mkdir -p heist-project/plans

touch heist-project/vault/gold.txt

touch heist-project/plans/strategy.conf

 

### Create Group

sudo groupadd planners

 

### Recursive Ownership Change

sudo chown -R professor:planners heist-project/

 

### Verification

ls -lR heist-project/

 

 

 

## Task 6: Practice Challenge (Scenario-Based)

 

### Users Created

sudo useradd tokyo

sudo useradd berlin

sudo useradd nairobi

 

### Groups Created

sudo groupadd vault-team

sudo groupadd tech-team

 

### Directory and Files Created

mkdir bank-heist

touch bank-heist/access-codes.txt

touch bank-heist/blueprints.pdf

touch bank-heist/escape-plan.txt

 

### Ownership Assignment

sudo chown tokyo:vault-team bank-heist/access-codes.txt

sudo chown berlin:tech-team bank-heist/blueprints.pdf

sudo chown nairobi:vault-team bank-heist/escape-plan.txt

 

### Final Verification

ls -l bank-heist/

 

 

 

## What I Learned

 

·        File ownership directly affects access control and security.

·        chown can modify both owner and group, while chgrp changes only group.

·        Recursive ownership (-R) is critical for managing application directories and logs.
