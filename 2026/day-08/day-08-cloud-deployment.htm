## Day 08 – Cloud Server Setup: Docker, Nginx & Web Deployment

 

## Objective

Deploy a real web server on the cloud, manage it via SSH, configure network access, and extract service logs — simulating real-world DevOps production tasks.

 

---

 

## Cloud Provider

- AWS EC2 (Free Tier) 

  *(Ubuntu 24.04 LTS)*

---

 

## Commands Used

 

1.  Connect to EC2 via SSH

chmod 400 devops-key.pem

ssh -i lab-8-practice-key.pem ubuntu@35.89.25.249

    

 

2.  Update System Packages

sudo apt update && sudo apt upgrade -y

 

3.  Install Nginx

sudo apt install nginx –y

 

4.  Verify Nginx Status

systemctl status nginx

 

5.  Enable Nginx on Boot

sudo systemctl enable nginx

 

6.  Check Open Port

sudo ss -tulpn | grep nginx

 

     ## Security Group Configuration

 

Inbound rules configured:

·        SSH → Port 22 (My IP)

·        HTTP → Port 80 (0.0.0.0/0)

## Nginx Log Management

1.  View Logs

sudo cat /var/log/nginx/access.log

sudo cat /var/log/nginx/error.log

 

2.  Save Logs to File

sudo cat /var/log/nginx/access.log > ~/nginx-logs.txt

 

3.  Verify Log File

ls -lh nginx-logs.txt

cat nginx-logs.txt

 

 

4.  Download Logs to Local Machine

scp -i devops-key.pem ubuntu@35.89.25.249:~/nginx-logs.txt

 

 

## Challenges Faced

 

·        Nginx page not accessible initially

o   Root cause: HTTP (port 80) not allowed in Security Group

o   Fix: Added inbound rule for port 80

 

·        Permission denied while accessing logs

o   Fix: Used sudo to read logs from /var/log/nginx
