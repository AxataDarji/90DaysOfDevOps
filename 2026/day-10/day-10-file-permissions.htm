Task 3: Understand Permissions

 

Permission Format:

rwxrwxrwx

│ │ │

│ │ └── Others

│ └──── Group

└────── Owner

 

Permission Value

Read (r)   4

Write (w)  2

Execute(x) 1

 

Current Permissions:

ls -l devops.txt notes.txt script.sh

 

Output:

 

Access Explanation:

 

·        Owner: read & write

·        Group: read only

·        Others: read only

·        No execute permission for any file

 

Task 4: Modify Permissions

 

1. Make script.sh Executable

chmod +x script.sh

./script.sh

 

Output:

Hello

 

2. Make devops.txt Read-Only for All

chmod a-w devops.txt

 

Permissions:

-r--r--r-- devops.txt

 

3. Set notes.txt to 640

chmod 640 notes.txt

 

Permissions:

-rw-r----- notes.txt

 

4. Create Directory project/ with 755

mkdir project

chmod 755 project

 

Permissions:

drwxr-xr-x project/

## What I Learned

 

1.  Linux permissions control who can read, write, or execute files and directories

2.  Numeric and symbolic chmod both achieve the same goal in different ways

3.  Execute permission is mandatory to run scripts, even if read access exists
