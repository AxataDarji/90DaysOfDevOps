2. Processes & Services Review (Day 04 / Day 05)

 

### Commands Re-run

·        ps aux | head

·        systemctl status ssh

·        journalctl -u ssh --no-pager | tail

 

### Observations

·        ps aux helps quickly identify running processes and CPU/memory usage

·        systemctl status gives instant insight into service state and recent logs

·        journalctl -u is powerful for debugging service failures

 

3. File Skills Practice (Days 06–11)

 

### Commands Practiced

echo "Revision check" >> revision.txt

chmod 640 revision.txt

chown ubuntu:ubuntu revision.txt

ls -l revision.txt

mkdir backup && cp revision.txt backup/

 

### Key Takeaways

 

·        Always verify permissions with ls -l

·        Use minimal required permissions to avoid security issues

·        Ownership matters as much as permission bits

 

4.  Cheat Sheet Refresh (Day 03)

 

Top 5 commands I’d reach for first during an incident:

 

·        ls -l – Check permissions & ownership

·        df -h – Disk space issues

·        free -m – Memory availability

·        systemctl status <service> – Service health

·        journalctl -xe – Error-level logs

 

5.  User / Group Sanity Check (Day 09 / Day 11)

 

Scenario Recreated

sudo useradd devtest

sudo passwd devtest

sudo chown devtest:devtest revision.txt

id devtest

ls -l revision.txt

 

Verification

·        id confirms UID, GID, and group membership

·        ls -l confirms correct ownership applied

 

6. Mini Self-Check

 

1.   Which 3 commands save you the most time right now, and why?

 

ls -l → instant permission & ownership clarity

systemctl status → fastest way to check service health

journalctl -u <service> → direct visibility into service logs

 

2.  How do you check if a service is healthy?

 

systemctl status nginx

journalctl -u nginx

ps aux | grep nginx

 

3.   How do you safely change ownership and permissions?

 

sudo chown ubuntu:ubuntu app.log

chmod 640 app.log

 

 

·        Always verify with ls -l

·        Avoid 777 unless absolutely necessary
